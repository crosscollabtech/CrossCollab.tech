Control room: how it works

1. Submit

Ministries post a need. We seed a GitHub repo with templates and labels.

2. Match

Contributors filter by skills and time. Good‑first‑issues prioritized.

3. Build

Open PRs. Clear milestones. Transparent progress and impact.

Submit a project

Pick the path that fits your need—quick service requests or full build projects.

Simple request

Service / setup help

Examples: clean up a donor list, set up livestreaming, configure forms, or organize existing tools. No code repo needed.

Deliverables: checklists, call notes, configs, or training docs.

Build a product

Code & deploy

Examples: launch a ministry site, build a web/app experience, or create a data pipeline. We can seed or link a GitHub repo with issues and labels.

Deliverables: repository, issues, milestones, and deployment plan.
